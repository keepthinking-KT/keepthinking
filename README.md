# KeepThinking — 让 AI 真正认识你的项目

我是一名独立开发者。

每天跟好几个 AI 助手打交道——Claude 写架构，Cursor 撸代码，Windsurf 做重构。它们都是好帮手，但有个共同的毛病：**每次新开对话，它们就像失忆了一样。**

第三天它不记得第一天定下的架构。改了十遍的 Bug，它又丢给你第一版的错误方案。Token 越烧越多，反复解释让人崩溃。

我找遍了市面上的方案，没有满意的。于是关起门写了这个——

## 🧠 KeepThinking

一个跑在你本地的 AI 记忆引擎。它默默记录你的每一次关键决策、每一次 Bug 修复、每一次架构变更。然后，在任何 AI 工具的新会话里，自动帮你「记起来」。

## 它做了什么

- **跨会话记忆**：/new 重启后自动注入过去的决策和经验，AI 无缝衔接
- **语义搜索**：用本地 ONNX 向量引擎理解你的自然语言，搜「部署不上线」能匹配到「CI/CD 配置」「Nginx 设置」
- **认知图谱**：自动把零散的决策连成知识网络，项目越大越有价值
- **Git 自动同步**：每次 commit 自动记录，AI 能「看懂」你的代码库演变
- **Bug 模式诊断**：识别 6 种常见 Bug 模式，给出修复建议
- **MCP 通用协议**：一行配置接入 Claude Desktop、Cursor、VS Code 等主流工具

## 为什么不一样

- **100% 本地**：所有数据在你硬盘上，零上传、不联网、不采集。你的项目记忆，你完全掌控
- **真正免费**：MIT 开源协议，没有 Pro 版本，没有收费计划
- **扔进去就能用**：一条命令安装，剩下的它自动处理

## 一分钟上手

```bash
curl -sL https://keepthinking.vip/install.sh | bash
```

安装后访问 `localhost:3456`，设置你自己的访问密码，可视化认知图谱就在眼前。
